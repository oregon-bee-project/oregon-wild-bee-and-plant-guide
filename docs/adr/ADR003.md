# ADR-003: Use Github Pages and Render to deploy the web application

**Status:** Accepted

**Context:**
We needed a way to deploy our application for use by others. Since we have different coding languages for our frontend and backend, we need two different deployments.

**Decision:**
We will use Github Pages to deploy the static front end, and Render to deploy the backend

**Options Considered:**

- AWS
- Google Domain
- Github Pages
- Render
- Do nothing

**Rationale:**
The combination of github pages and render provides us with an easy to use deployment strategy with lots of existing documentation. It is also free.

**Consequences:**

- Team members need to learn the basic deployment and unit test implementation.
- There is a small spinup time when there is no activity on the webpage.
- Future migrations may require careful planning.
- Incoming changes will have to pass the CI/CD tests.

- [Render documentation](https://render.com/docs)
